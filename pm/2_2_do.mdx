**B8 — recommend reclassifying as won't-fix / by-design.** The two calls are **required** by §8.0's
before/after **content-preservation proof** — a single call cannot prove the output still matches the
input. Post-B1 the cost is **~1 MB × 2** (was ~105 MB × 2), which is no longer a memory defect.

**B10 — what was measured.** Downscaled decode vs. a **replicated pre-B1 full-res decode**: Hamming
**0 (identical)** at 1024×768 / 2000×1500 / 4000×3000 / 6000×4000 and at under-edge sizes; **2 bits** at
noise ±127; **8 bits** only on **pure random noise** (quality 11 — below the matching gate). Path vs.
Buffer = **bit-identical**. **B1 did not materially change fingerprints.**

> **B10's premise was vacuous on this machine.** An investigation found **5,386 sidecar YAMLs on disk
> with ZERO populated fingerprints** (all `fingerprint: null`). Fingerprints only materialize inside
> `compressFile()`, **which has never run on this machine** — so the "corpus we must not silently
> invalidate" doesn't exist here and B1's real-world risk was **theoretical**.
>
> **Caveat.** The pre-existing spec's `syntheticPhoto` oracle is **smooth** and scores 0 regardless of
> whether the downscale is sound — **it would pass even if B1 were broken**. The **noise case is the
> discriminating one**.

**B9 was the trap — and B1 sprang it.** The reasoning was: `estimatedBytes()` is
`min(fileSize, INLINE_MAX_BYTES)` — correct for describe (bytes uploaded), **catastrophic for
compress**, where a 5 MB JPEG decodes to ~96 MB, so the budget would **under-count ~20×** and admit a
fatal batch while reporting healthy. That was true **of the pre-B1 code**.

**B1 removed the premise.** With `decodeForHash()` resizing to 512px **before** the raw decode, the
compress bucket holds **~1 MB** in this heap; everything else on its path is subprocess RSS. So
`estimatedBytes()`'s `if (t.op !== "describe") return 0` is **correct**, and implementing B9 as written
would now **over-reserve ~96×** — throttling the bucket for bytes it never allocates. **B9 is closed as
obsolete**; [to_fix.mdx §7.4](./to_fix.mdx) carries the full RESOLUTION and the amended invariant.

**The dependency is load-bearing:** this only holds while B1 stays landed. Revert the downscale and
B9 comes back. That is precisely what B11/§3.4's "never downscale governs the *output*, not the
fingerprint's internal decode" note exists to prevent.
