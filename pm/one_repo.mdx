# LargeFileBridge — One repo

The **One repo** screen is where the work happens. You reach it by clicking a repo on the
[Repos](./repos.mdx) page; it opens that repo at `/repos/$repoId`. It has two jobs:

1. **Show this repo's settings, progress, and status** — is per-repo sync on? are the files pinned?
   which of your computers hold them? when did it last sync? — and let you **Sync now**.
2. **Let you decide, file by file, what LFBridge should move** — the core interaction. For every
   large git-ignored file the repo contains, you choose **Sync**, **Ignore**, or leave it
   **Undecided**. Most of the user's time here is spent scanning "how many will sync, how many will
   not, and which ones I still need to decide on."

This spec builds on the siblings and does not repeat them:

* [Repos](./repos.mdx) — the list this screen drills into; defines the LOCKED vocabulary
  **Repo → Large file → Decision** (§1 there), the `RepoStatus` enum, and the per-repo counts.
* [Left bar](./left_bar.mdx) — the left navigation (source of truth `pm/left_bar.yaml`).
* [README](../README.md) — the product charter (git for small files; IPFS/p2p for the big ones).

## 1. The decision model (LOCKED)

Every large file in the repo is in exactly **one** of three **decision** states. This is the
vocabulary the whole app shares:

| Decision | Meaning | LFBridge behavior |
| --- | --- | --- |
| **Sync** | The user wants this file kept in sync across their computers. | LFBridge adds it to the manifest, pins it on IPFS, and transfers it to/from peers. |
| **Ignore** | The user deliberately does **not** want this file synced. | LFBridge leaves it alone — never added to the manifest, never transferred. Stays a purely local file. |
| **Undecided** | LFBridge found the file but the user has not chosen yet. | LFBridge does **nothing** with the bytes yet; it just surfaces the file so the user can triage it. This is the queue of decisions to make. |

Independently of the decision, a **Sync** file has a **transfer status** — where its bytes are
relative to the network (§4.2). A file's decision is user intent; its transfer status is reality.

```ts
// One row of the files table.
export interface FileRow {
  fileId: string;        // stable id (repoId + relative path)
  path: string;          // path relative to the repo root
  sizeBytes: number;     // on-disk (or expected) size
  cid: string | null;    // IPFS content id once added to the manifest; null while Undecided/Ignore
  decision: "sync" | "ignore" | "undecided";
  transfer: TransferStatus;   // meaningful for decision === "sync" (see §4.2)
  peers: string[];       // ids of your computers currently pinning this file
  changedAt: string;     // ISO date-time the file last changed on disk
}

export type TransferStatus =
  | "synced"     // present locally AND pinned on >=1 peer
  | "pending"    // queued to push out or fetch in
  | "fetching"   // transfer in progress (inbound)
  | "pushing"    // transfer in progress (outbound)
  | "missing"    // in the manifest but the bytes are not on this machine
  | "error"      // last transfer failed
  | "na";        // not applicable (decision is ignore/undecided)
```

## 3. Header + status/settings strip

### 3.1 Header

* **Breadcrumb** — `‹ Repos / <repo name>`; the repo name is the title, the absolute **path** shown
  muted beneath it.
* **Sync now** button (top-right, `RefreshCw`) — the repo's one **primary** action: push out every
  `Sync` file this machine has that peers lack, fetch every `Sync` file peers have that this machine
  lacks. States: idle (`Sync now`) → running (`Syncing…`, spinner, disabled) → done (counts + table
  refresh). It never changes any decision.
* **`⋮` "more" menu** (top-right, next to Sync now) — because this page **is one repo**, it carries the
  **repo entity menu**: the same catalog as a Repos-table row's kebab
  ([menus.mdx §4 / §5.1](./menus.mdx#4-the-entity--more-menu--single-entity-pages)) — Open in File
  System · View pinned on IPFS · Rescan · Sync now · Git-ignore big files… · Toggle Synced · Repo
  settings… · Copy path · **Remove repo (unregister)** (destructive, last, never deletes local bytes).
  *Open repo* is omitted (you are already here). Sync now appears both as the primary button and in the
  menu so the menu is a complete catalog.

### 3.2 Status / settings strip

A single horizontal strip of the repo's live status (read) and its one per-repo setting:

| Field | Meaning |
| --- | --- |
| **Sync** (toggle) | Per-repo on/off. When **off**, LFBridge skips this repo during background syncs; decisions are preserved but no bytes move. The only editable control in the strip. |
| **Status** | The rolled-up `RepoStatus` pill ([Repos §4.2](./repos.mdx)) — up to date / syncing / behind / needs review / error / never. |
| **Peers** | Count of your computers currently pinning this repo → clicking opens [Peers](./left_bar.mdx) filtered to this repo (peers page is out of scope here). |
| **Last sync** | Relative time of the last completed sync (absolute on hover). |
| **IPFS** | The local IPFS node health — `ok` (green) / `unreachable` (red). If unreachable, Sync now is disabled with a tooltip. |

Note: the app-wide **background sync process** (installed? on/off? the every-15-min `launchd`
schedule) is the **Sync** left-bar page, not this strip. This strip's toggle is only *this repo's*
participation in it.

## 4. The files table (TanStack, house style)

**One row per large git-ignored file** in the repo. Same flat `DataTable` and the same standard
control row as [Repos §3](./repos.mdx) — including **header-click sort** (click a column header to
sort; click again to toggle asc ↔ desc; [Repos §3.1](./repos.mdx)).

**Height — bounded (the content-below exception).** Unlike Repos, this page renders content **beneath**
the files table — the **Repo details** disclosure (§3.2 status strip, now a click-away disclosure at
the bottom of the page). Because something sits **below** the table, it is the **exception** to the
site-wide full-page-height rule ([Repos §3.3.1](./repos.mdx), [charter Tables rule](../CLAUDE.md)): the
table keeps a **bounded** height (it does **not** expand to fill the viewport) so the Repo-details
disclosure below it stays visible. In code this is `fillHeight={false}` on the `DataTable`.

### 4.1 Summary counts (above the control row)

A one-line summary the user reads first: **`128 Sync · 5 Undecided · 40 Ignore`**. Each segment is a
**quick filter** — clicking `Undecided` filters the table to the undecided files (the triage queue),
etc. `Undecided` is accent-highlighted whenever it is > 0.

### 4.2 Control row — LOCKED

Identical to the standard toolbar ([Repos §3.1](./repos.mdx)):

* **Search box** (left) — substring, case-insensitive, over the **file path**.
* **Filter icon** (`⛛`, icon-only, no box) — dropdown of filterable columns; operators by column
  kind; multiple filters ANDed; colored when active; resets to page 1.
* **Sort icon** (icon-only, no box) — dropdown of sortable columns, asc/desc; colored when active.
* **Bulk menu** (`⋮`) — enabled when ≥1 row is checkbox-selected (§4.4).

### 4.3 Columns

| # | Column | Kind | Sort | Filter | Notes |
| --- | --- | --- | --- | --- | --- |
| 0 | **☐ select** | — | — | — | Row checkbox; header checkbox selects the current page (for bulk decisions). |
| 1 | **File** | text | ✓ | ✓ (contains) | Path relative to the repo root; directory prefix muted, filename bold. |
| 2 | **Size** | int (bytes) | ✓ | ✓ | Human-readable (`8.2 GB`), right-aligned; sorts/filters on raw bytes. |
| 3 | **Decision** | enum | ✓ | ✓ | A **`[Sync ▾]` inline select** — Sync / Ignore / Undecided (§4.5). `filterOptions` = the three values. |
| 4 | **Status** | enum | ✓ | ✓ | `transfer` status pill (§4.6); `—` for non-sync files. |
| 5 | **Peers** | int | ✓ | ✓ | How many of your computers hold this file. `0` on a **Sync** file renders red (not backed up yet). |
| 6 | **CID** | text | ✓ | ✓ | IPFS content id (monospace, middle-truncated, click to copy); blank until the file is a Sync file with a CID. |
| 7 | **Changed** | timestamp | ✓ | ✓ | `changedAt`, relative with absolute tooltip. |
| — | **`⋮`** | — | — | — | The row **kebab** — the **file** action catalog ([menus.mdx §5.3](./menus.mdx#53-file)). A control cell: opens the menu, never navigates. Right-click on the row opens the same menu ([menus.mdx §3.1](./menus.mdx#31-right-click-opens-the-same-menu-locked)). |

Pagination: **default 500** ([Repos §3.3](./repos.mdx)); the **file count** shows left of the pager.

### 4.7 Cell navigation (per file) — LOCKED

Beyond the inline Decision select, each data cell is a deep link
([menus.mdx §2](./menus.mdx#2-cell-navigation--clicking-a-cell-is-a-deep-link)):

| Cell | Click target |
| --- | --- |
| **File** | The **View one file** page ([files.mdx](./files.mdx)) for this file. |
| **Size** | View one file (details). |
| **Decision** | Inline select — edits in place, does not navigate (§4.5). |
| **Status** | View one file (its transfer detail). |
| **Peers** | [Peers](./left_bar.mdx) filtered to this file (which computers hold it). |
| **CID** | **Copies** the CID (click-to-copy); an inline affordance opens [IPFS](./ipfs.mdx) filtered to it. Does not navigate away. |
| **Changed** | View one file. |
| **`⋮`** | Opens the **file action menu** — never navigates (§4.3). |

The **File** row's kebab (and right-click) is the **file** catalog ([menus.mdx §5.3](./menus.mdx#53-file)):
View file · Open containing folder · Add to IPFS / Remove from IPFS · Set decision ▸ · Sync now ·
Compress… · Convert to compressible… · Git-ignore this file · Find duplicates (fingerprint) · the two
sticky flags **Never IPFS** / **Do not compress** ([menus.mdx §6.6](./menus.mdx#66-the-two-sticky-flags--never-ipfs--do-not-compress-locked)) ·
View on IPFS · Copy CID / path · **Stop syncing (untrack/unpin)** (destructive, never deletes local
bytes). Compress/convert/ignore are **explicit-click offers with a confirm step**
([menus.mdx §6.1](./menus.mdx#61-never-act-on-files-without-an-explicit-click)).

### 4.4 Bulk actions

When rows are selected, the `⋮` bulk menu applies one decision to all of them at once — the fast
path for triaging many files:

* **Set to Sync** · **Set to Ignore** · **Set to Undecided** — bulk-writes the decision.
* **Sync now (selected)** — queues transfers for the selected `Sync` files.

Selecting the header checkbox selects the whole current page; a "Select all N matching" affordance
appears when a filter is active, to act on the entire filtered set, not just the page.

### 4.5 The Decision select (per row) — the core interaction

Each row's **Decision** cell is an inline select with the three states. Changing it writes
immediately (optimistic, with rollback on error):

* → **Sync**: LFBridge adds the file to the manifest, computes/records its **CID**, pins it, and
  queues transfers to peers. The row's Status moves to `pending` → `fetching`/`pushing` → `synced`.
* → **Ignore**: removes it from the manifest (if present); Status becomes `—`; bytes stay local only.
* → **Undecided**: parks it back in the triage queue; no bytes move.

Changing a decision never deletes the user's local file — it only changes what LFBridge *does* with
it.

### 4.6 Transfer status pill (LOCKED)

| `transfer` | Meaning | Pill |
| --- | --- | --- |
| `synced` | Present locally and pinned on ≥1 peer. | green |
| `pending` | Queued to push out or fetch in. | amber |
| `fetching` / `pushing` | Transfer in progress. | blue (accent), animated |
| `missing` | In the manifest but bytes not on this machine (fetch to restore). | red outline |
| `error` | Last transfer failed. | red |
| `na` | Decision is Ignore/Undecided — nothing to transfer. | `—`, muted |

## 5. Empty & loading states

* **No large files:** the table is replaced by a card — "No large files found in this repo. LFBridge
  looks for git-ignored files under `large/` and the ignored media extensions."
* **All decided:** the summary shows `Undecided 0`; the quick-filter for Undecided is disabled.
* **Loading:** skeleton rows sized to the columns; no layout shift.
* **IPFS node unreachable:** a banner at the top of the content; Sync now and per-row transfers are
  disabled, but decisions can still be edited (they queue for when the node returns).

   `‹ Repos / <name>` breadcrumb, the absolute path, and a **Sync now** button that syncs only this

   Peers, Last sync, and IPFS node health; when the node is unreachable, Sync now is disabled.

   Undecided is accent-highlighted when > 0.
4. The **files table** is one TanStack table, one row per large git-ignored file, flat/chromeless,
   with the standard control row (search left; icon-only filter `⛛` and sort dropdowns right) and
   columns: select, File, Size, Decision, Status, Peers, CID, Changed, and a trailing **`⋮`** kebab.
5. **Pagination defaults to 500**; the **file count** shows left of the pager.
6. The **Decision** cell is a per-row inline select (Sync / Ignore / Undecided) that writes
   immediately per §4.5; changing a decision never deletes the user's local file.
7. **Bulk actions** (`⋮`, enabled on selection) set Sync / Ignore / Undecided across selected rows
   and can Sync-now the selected Sync files; the header checkbox selects the page, with a
   "select all matching" affordance under an active filter.
8. The **Status** column follows the LOCKED transfer-status pill mapping (§4.6); a Sync file with
   Peers = 0 renders the peer count red.
9. Empty / all-decided / loading / node-unreachable states render per §5.
10. The header carries a top-right **`⋮` "more" menu** = the **repo** entity catalog
    ([menus.mdx §5.1](./menus.mdx#51-repo)), with *Open repo* omitted and *Sync now* also present as the

    Changed → View one file; Peers → Peers filtered to the file; CID copies); the row **`⋮` kebab** (and
