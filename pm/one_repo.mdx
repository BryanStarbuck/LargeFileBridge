# LargeFileBridge вҖ” One repo

The **One repo** screen is where the work happens. You reach it by clicking a repo on the
[Repos](./repos.mdx) page; it opens that repo at `/repos/$repoId`. It has two jobs:

1. **Show this repo's settings, progress, and status** вҖ” is per-repo pinning on? are the files pinned?
   which of your computers hold them? when did it last pin? вҖ” and let you **Pin now**.
2. **Let you decide, file by file, what LFBridge should move** вҖ” the core interaction. For every
   large git-ignored file the repo contains, you choose **Add to IPFS (pin)** (the `sync` decision
   value), **Ignore**, or leave it **Undecided**. Most of the user's time here is spent scanning
   "how many will pin, how many will not, and which ones I still need to decide on."

The One-repo screen is rendered in code:
`~/BGit/Bryan_git/LargeFileBridge/code/packages/frontend/src/pages/repos/OneRepoPage.tsx` вҖ” `OneRepoPage` component

This spec builds on the siblings and does not repeat them:

* [Task tabs](./task_tabs.mdx) вҖ” the **All / IPFS / Compress / Transcribe / AI descriptions** tab strip this
  screen's header now carries (В§3.1 here). Selecting a tab re-tunes the metrics strip, the visible table
  columns, the default sort, and the row filter; that whole system вҖ” including the metrics panels, the
  hover-info region, the per-metric popup + blue header-primary behavior, and the three-state
  Transcribe / Compress / Describe status icons вҖ” is defined there and only summarized here.
* [Repos](./repos.mdx) вҖ” the list this screen drills into; defines the LOCKED vocabulary
  **Repo вҶ’ Large file вҶ’ Decision** (В§1 there), the `RepoStatus` enum, and the per-repo counts.
* [pin_process.mdx](./pin_process.mdx) вҖ” the **pin process** the Pin-now button triggers:
  every pin pass is a **full pass**, a repo trigger sets **priority** (not scope), the pass is parallel,
  and it is never a silent no-op. The button is one trigger of that pass.
* [Left bar](./left_bar.mdx) вҖ” the left navigation (source of truth `pm/left_bar.yaml`).
* [README](../README.md) вҖ” the product charter (git for small files; IPFS/p2p for the big ones).

## 1. The decision model (LOCKED)

Every large file in the repo is in exactly **one** of three **decision** states. This is the
vocabulary the whole app shares:

| Decision | Meaning | LFBridge behavior |
| --- | --- | --- |
| **Add to IPFS (pin)** | The user wants this file kept pinned across their computers. Stored decision value: `sync`. | LFBridge adds it to the manifest, pins it on IPFS, and transfers it to/from peers. |
| **Ignore** | The user deliberately does **not** want this file pinned. | LFBridge leaves it alone вҖ” never added to the manifest, never transferred. Stays a purely local file. |
| **Undecided** | LFBridge found the file but the user has not chosen yet. | LFBridge does **nothing** with the bytes yet; it just surfaces the file so the user can triage it. This is the queue of decisions to make. |

Independently of the decision, an **Add-to-IPFS (pin)** file (decision value `sync`) has a **transfer
status** вҖ” where its bytes are relative to the network (В§4.2). A file's decision is user intent; its
transfer status is reality.

> **This table is the IPFS axis only вҖ” one of two.** The full decision the user makes has **two
> independent axes** вҖ” **Add to IPFS (pin)** *and* **Add to git ignore** вҖ” each on/off separately, and it
> is recorded (with who/when/the Storage ID) in a **repo-level, team-shared** ledger so a whole team
> shares one set of decisions. That is specified end to end in **[decisions.mdx](./decisions.mdx)**. The
> three-state enum here is the **frozen IPFS-axis projection** of that ledger (`ipfs:true вҶ’ sync`,
> `ipfs:false вҶ’ ignore`, no record вҶ’ undecided) вҖ” this screen keeps rendering it verbatim; the wire value
> `sync` is never renamed. The git-ignore axis is applied through [git_ignore.mdx](./git_ignore.mdx).

```ts
// One row of the files table.
export interface FileRow {
  fileId: string;        // stable id (repoId + relative path)
  path: string;          // path relative to the repo root
  sizeBytes: number;     // on-disk (or expected) size
  cid: string | null;    // IPFS content id once added to the manifest; null while Undecided/Ignore
  decision: "sync" | "ignore" | "undecided";  // FROZEN wire value; "sync" renders as "Add to IPFS (pin)" in the UI
  transfer: TransferStatus;   // meaningful for decision === "sync" (see В§4.2)
  peers: string[];       // ids of your computers currently pinning this file
  changedAt: string;     // ISO date-time the file last changed on disk
}

export type TransferStatus =
  | "pinned"     // present locally AND pinned on >=1 peer
  | "pending"    // queued to push out or fetch in
  | "fetching"   // transfer in progress (inbound)
  | "pushing"    // transfer in progress (outbound)
  | "missing"    // in the manifest but the bytes are not on this machine
  | "error"      // last transfer failed
  | "na";        // not applicable (decision is ignore/undecided)
```

### 3.2 Status / settings strip

A single horizontal strip of the repo's live status (read) and its one per-repo setting:

| Field | Meaning |
| --- | --- |
| **Pin** (toggle) | This repo's **IPFS-pinning opt-in** вҖ” *not* a "managed vs. unmanaged" switch. ### 4.5 The Decision select (per row) вҖ” the core interaction

Each row's **Decision** cell is an inline select with the three states. Changing it writes
immediately (optimistic, with rollback on error):

* вҶ’ **Add to IPFS (pin)** (decision value `sync`): LFBridge adds the file to the manifest,
  computes/records its **CID**, pins it, and queues transfers to peers. The row's Status moves to
  `pending` вҶ’ `fetching`/`pushing` вҶ’ `pinned`.
* вҶ’ **Ignore**: removes it from the manifest (if present); Status becomes `вҖ”`; bytes stay local only.
* вҶ’ **Undecided**: parks it back in the triage queue; no bytes move.

Changing a decision never deletes the user's local file вҖ” it only changes what LFBridge *does* with
it.

Writing a per-file decision (single or bulk) to the repo's `config.yaml` is handled in code:
`~/BGit/Bryan_git/LargeFileBridge/code/packages/backend/src/modules/repos/repos.router.ts` вҖ” `reposRouter` вҶ’ `PATCH /api/repos/:repoId/files`
