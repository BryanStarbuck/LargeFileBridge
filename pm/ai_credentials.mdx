# LargeFileBridge — AI credentials (Gemini key) setup

`LargeFileBridge` generates an AI description of an image/video by uploading it to a vision provider
([ai_description.mdx](./ai_description.mdx)). That needs an **API key**. **Gemini** (Google) is the only
provider that describes **video**, so a Gemini key is the one that matters most. This spec is the
authoritative source for **what the user sees when no key is configured**, and **where a key may come
from**.

Siblings this builds on and does not repeat:

* [ai_description.mdx](./ai_description.mdx) — the description feature itself (adapters, prompts, storage,
  the one-deliberate-upload posture). This file is authoritative for the **credential resolution order**
  and the **missing-credentials UX**; that file references this one.
* [media_viewer.mdx §6](./media_viewer.mdx) — the viewer band that hosts the **Generate description**
  button which triggers the popup below.

---

## 1. Credential resolution — three sources, in order (LOCKED)

The Gemini key is resolved from **three** sources, checked in this exact order; the first that yields a
usable key wins:

1. **App config** — `ai.gemini.api_key` in the local `config.yaml` (written by **Settings → Tools**).
2. **Environment variables** — `GEMINI_API_KEY` / `GOOGLE_GENAI_API_KEY` / `GOOGLE_API_KEY`.
3. **The shared Google/Gemini key file** — `~/.config/GoogleCloud/apikey.yaml`. This is the **SAME file**
   the `~/BGit/all/tools` Gemini / "nano banana" image tools already read. It has two fields; we prefer
   **`apiKey`**, then fall back to **`4k_apiKey`** (matching the tools' vision path,
   `~/BGit/all/tools/Img_to_Description/img_describe.js`). A machine already set up for those tools
   therefore describes video/images with **no extra setup**.

The third source is the important addition: before it, a user with a working `apikey.yaml` still saw
"no provider configured." Reading that file closes the gap between this app and the sibling CLI tools.

Grok (`XAI_API_KEY` / `GROK_API_KEY`) and OpenAI (`OPENAI_API_KEY`) resolve from config → env only (no
shared file); they describe **images only**.

Resolving the Gemini key from the shared GoogleCloud key file (prefer `apiKey`, else `4k_apiKey`; tolerate
absent/placeholder/invalid) is handled in code:
`~/BGit/Bryan_git/LargeFileBridge/code/packages/backend/src/config/google-apikey-file.ts` — `loadGoogleApiKey()` / `googleApiKeyFileInfo()`.

Chaining the three sources for Gemini (config → env → shared file) is handled in code:
`~/BGit/Bryan_git/LargeFileBridge/code/packages/backend/src/modules/describe/adapters.ts` — the `gemini` adapter's `apiKey()`, and `providerKeySources()` (adds a `usingFile` source flag).

## 2. The credentials-missing popup (Generate → Close / Instructions)

When the user clicks **Generate description** and **no** vision key resolves, the app does **not**
dead-end on a disabled button. The button stays **clickable**, and clicking opens a **popup**:

```
┌───────────────────────────────────────────────┐
│ 🔑  AI credentials not found                   │
│                                                │
│ Large File Bridge couldn't find an AI provider │
│ key on this computer, so it can't generate a   │
│ description yet.                                │
│                                                │
│ Generating a description needs a Gemini API    │
│ key (the only provider that describes video).  │
│ The credentials file that holds it wasn't      │
│ found. Open the instructions to see exactly    │
│ where the file goes, its format, and what to   │
│ put inside.                                     │
│                                                │
│                       [ Close ] [ Instructions ]│
└───────────────────────────────────────────────┘
```

* **Close** — dismisses the popup; the user stays on the viewer.
* **Instructions** — opens the **AI credentials setup page in a NEW TAB** (`/ai-credentials`, §3) so the
  viewer stays open while the user sets up the key.

The popup fires in **two** cases, both surfacing the same dialog:

* **Known up front** — the viewer's provider probe (`GET /api/describe/providers` → `anyAvailable:false`)
  already says no key exists; clicking Generate opens the popup immediately (no wasted network call).
* **At runtime** — a generate call comes back `no_provider` (e.g. a **video** on a machine that has only
  image-capable Grok/OpenAI keys — video needs Gemini); the honest reason from the backend is shown in
  the popup.

The popup (Close / Instructions; Instructions opens `/ai-credentials` in a new tab) is handled in code:
`~/BGit/Bryan_git/LargeFileBridge/code/packages/frontend/src/components/describe/CredentialsMissingDialog.tsx` — `CredentialsMissingDialog`.

Deciding when to open the popup (known-no-key up front, or a runtime `no_provider`) is handled in code:
`~/BGit/Bryan_git/LargeFileBridge/code/packages/frontend/src/pages/entity/MediaViewer.tsx` — `MediaAnalysis` (`onGenerateDescription`), and `~/BGit/Bryan_git/LargeFileBridge/code/packages/frontend/src/lib/describe.ts` — `runDescribeFile` (`onNoProvider` callback).

## 3. The Instructions page (`/ai-credentials`) — where the file goes, its format, the placeholder

The Instructions button opens a **full page** that walks the user through fixing it. It shows a live
status banner (ready vs. not), then the three sources, leading with the shared key file:

* **Step 1 — create the shared key file (recommended).** The exact **path**
  (`~/.config/GoogleCloud/apikey.yaml`) with a copy button, whether it already exists / is configured,
  and the **exact YAML shape** to paste (also copyable):

  ```yaml
  # Google Cloud / Gemini API key(s).
  # Get a key at https://aistudio.google.com/app/apikey
  apiKey: "YOUR_GEMINI_API_KEY"

  # Optional: a separate key for 4K image generation (Google Imagen). Used only as a
  # fallback for AI descriptions when apiKey above is absent.
  4k_apiKey: "YOUR_GEMINI_API_KEY"
  ```

  The page states the field priority (**`apiKey` first, then `4k_apiKey`**) and warns that this file holds
  a secret — keep it out of any git repository.
* **Other ways to provide a key** — **Settings → Tools** (writes into the app `config.yaml`, whose path
  the page prints) and **environment variables** (the exact var names per provider).

The page **never** shows a real key value — only the path and a **placeholder** schema. It is opened in a
new tab, but is a normal allow-list-gated app route, so it renders inside the app shell for the signed-in
user.

The full instructions page (status banner, the shared-file path + placeholder YAML with copy buttons, the
Settings/env alternatives) is handled in code:
`~/BGit/Bryan_git/LargeFileBridge/code/packages/frontend/src/pages/settings/AiCredentialsPage.tsx` — `AiCredentialsPage` (route `/ai-credentials` in `router.tsx`).

The data it renders (file path/filename/dir/fields/exists/configured + placeholder schema, the app
config.yaml path, and the per-provider env var names) comes from code:
`~/BGit/Bryan_git/LargeFileBridge/code/packages/backend/src/modules/describe/describe.service.ts` — `aiCredentialsInfo()`, exposed as `GET /api/describe/credentials` in `describe.router.ts`. No raw key value is ever returned.

### 3.1 Page actions — verify & reveal (proposed)

The instructions page (`/ai-credentials`) is a walkthrough, not a file list, so it carries no
action-links row ([page_actions.mdx §0](./page_actions.mdx)). Two **header buttons** would close the loop
after the user pastes their key:

| Action | What it does | Placement | Status |
| --- | --- | --- | --- |
| **Re-test key** | Re-runs the three-source resolution (config → env → shared file, §1) and flips the status banner ready ↔ not-ready — so the user pastes their key and confirms it resolves **without** reloading or going back to the viewer. Reads `GET /api/describe/credentials` again (never returns the key value, §4). | Header, right-aligned, primary. | **proposed** |
| **Reveal file location in Finder** | Opens Finder at `~/.config/GoogleCloud/` (the shared key file's directory, §3) so the user can create/edit `apikey.yaml` in place — the OS hand-off pattern of [os_open.mdx](./os_open.mdx). | Header, right-aligned. | **proposed** |

Neither is implemented yet, so no code pointer; when built they belong in the header of
`~/BGit/Bryan_git/LargeFileBridge/code/packages/frontend/src/pages/settings/AiCredentialsPage.tsx` —
`AiCredentialsPage`, re-reading `aiCredentialsInfo()` (Re-test) and driving the OS reveal (Reveal in
Finder). Both are page-level actions, never per-row `⋮` items.

## 4. Privacy posture (unchanged)

Reading the shared key file does **not** widen the charter's one-deliberate-upload posture
([ai_description.mdx §7](./ai_description.mdx)). The key is only ever used to make the **explicit,
user-initiated** Gemini upload on a Generate click. The key file is read locally; its value is **never**
written back into the repo and **never** returned to the frontend (the Instructions page and the popup
only ever see the path and a placeholder). Nothing here touches the local-only perceptual-fingerprint
feature, which must never phone home.

## 5. Acceptance criteria

1. The Gemini key resolves from **config → env → the shared `~/.config/GoogleCloud/apikey.yaml`** (prefer
   `apiKey`, else `4k_apiKey`), so a machine already configured for the `~/BGit/all/tools` Gemini tools
   generates descriptions with no extra setup. A missing / placeholder-only / invalid file is treated as
   "no key from here," never an error.
2. Clicking **Generate description** with no key resolvable opens a **popup** titled **"AI credentials not
   found"** with exactly two buttons — **Close** and **Instructions** — rather than a disabled button. The
   popup also fires on a runtime `no_provider` (e.g. a video with only image-capable keys).
3. **Instructions** opens `/ai-credentials` in a **new tab**: a full page giving the exact **file path**,
   the **YAML format**, and a **placeholder** (`apiKey` / `4k_apiKey`) to fill in — with copy buttons —
   plus the Settings and env-var alternatives. No real key value is ever shown.
4. The credential details come from `GET /api/describe/credentials` (`aiCredentialsInfo()`), allow-list
   gated, returning path/format/placeholder + the config.yaml path + env var names, never a raw key.
